Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 always fails to predict the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is no better than one that naively predicts the mean of the target variable.

Question 3 - Training and Testing¶

What is the benefit to splitting a dataset into some ratio of training and testing subsets for a learning algorithm?
Hint: What could go wrong with not having a way to test your model?

Answer:

Splitting the data into training and testing subsets provides and independent set of data that can be used to test the performance of the model. This is especially important to prevent the model from overfitting the data. As the model is trained, it adapts to reduce the error of predictions made on the training set. If the model is flexible enough, it is possible to train it to predict these data points with minimal error while failing to generalize to data points it has not seen. The performance of the model on the testing set will be more indicative of how the model will perform in the real world when applied to new data.

It is also important to note that the data was shuffled before it was split into the training and testing sets. This is important to minimize any bias in the data based on ordering. In this case, we used a fixed seed to shuffle the data. This makes the split deterministic so that we can repeat experiments on the data and get the same results each time.

Question 7 - Grid Search¶

What is the grid search technique and how it can be applied to optimize a learning algorithm?

Answer:

Grid search is a technique to systematically explore and fine tune several parameters of a model all at once. For example, we could consider models of different complexity along one axis of the grid, and then for each of these models try several different regularization parameters along another axis. Because these different parameters may be interrelated (e.g. a more complex model would benefit from more regularization), exploring all combinations of parameters in likely to result in better decisions than picking the best option from each axis in turn.

Question 8 - Cross-Validation¶

What is the k-fold cross-validation training technique? What benefit does this technique provide for grid search when optimizing a model?
Hint: Much like the reasoning behind having a testing set, what could go wrong with using grid search without a cross-validated set?

Answer:

K-fold cross-validation is a technique that serves a similar purpose as training and testing splits. However, instead of dividing data into just two parts, the data is divided into K equally sized bins. Several models are then trained, each using a different bin as a hold out to score the results. The results are then averaged for a final score. This maximizes the data available for training while still allowing the models to be tested with data not used in their training to minimize overfitting.

Using cross-validation instead of simple training and test sets is especially important when using grid search. If each of the combinations of parameters in a grid search were validated by testing against the same test set, the winning combination of parameters would be the ones that best fit the test data and not necessarily the ones that best generalized against unknown values. Cross-validation helps prevent overfitting against a single test set.

# TODO: Import 'make_scorer', 'DecisionTreeRegressor', and 'GridSearchCV'
from sklearn.grid_search import GridSearchCV
from sklearn.metrics import make_scorer
from sklearn.tree import DecisionTreeRegressor

def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 0)

    # TODO: Create a decision tree regressor object
    regressor = DecisionTreeRegressor()

    # TODO: Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = {'max_depth': [x+1 for x in range(10)]}

    # TODO: Transform 'performance_metric' into a scoring function using 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # TODO: Create the grid search object
    grid = GridSearchCV(regressor, params, scoring=scoring_fnc, cv=cv_sets)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    # Return the optimal model after fitting the data
    return grid.best_estimator_
